# View Plan: Client Portal

## Views Covered
- Portal Login
- Portal Dashboard
- Portal Photos
- Portal Selections
- Portal Draws

---

# OVERVIEW

## Purpose
The Client Portal is a separate, simplified interface for clients to:
- View project progress (photos, schedule)
- Make selection choices on allowances
- View and approve draws
- Access project documents

## Access
- Separate URL: `/portal` or subdomain `portal.rossbuilt.com`
- Authentication via email/password (created by builder)
- Can be enabled/disabled per client
- Client sees only their job(s)

## Design Principles
- Clean, simple, client-friendly
- Mobile-first (clients often on phones)
- Read-mostly (limited input actions)
- No financial details beyond draws
- Professional branding (builder's logo/colors)

## Features
- View pending and past draws
- See schedule of values breakdown
- View photos from the period
- Download PDF version
- Approve draw (digital signature)
- Request changes/questions
- Running summary of contract

## Workflow
1. Builder creates draw in CMS
2. Builder submits for client review
3. Client receives email notification
4. Client reviews in portal
5. Client approves or requests changes
6. If approved â†’ draw moves to payment processing

## Access Control

### Builder Side
- Enable/disable portal per client
- Set client email/password
- Control which photos are shared
- Submit draws for approval
- Add selection options

### Client Side
- View only their job(s)
- Cannot see financial details beyond draws
- Cannot see vendor info
- Cannot see internal notes
- Read-only except for:
  - Selection confirmations
  - Draw approvals

---

## Affected By Changes To
- Jobs (portal shows job data)
- Clients (portal user linked to client)
- Photos (shared photos visible)
- Draws (pending draws for approval)
- Selections (pending selections for confirmation)
- Files (shared documents)
- Change Orders (visible for review)
- Warranties (accessible after completion)

## Affects
- Draws (client approval updates draw status)
- Selections (client confirmation updates selection)
- Activity logs (client actions recorded)

## Mobile Considerations

- **Client-focused design**: Mobile-first responsive layouts
- Simple login with "Remember Me" for return visits
- Dashboard with progress bar and recent activity
- **Photos**: Swipe gallery with pinch-to-zoom
- Download/share photos to camera roll
- Photo grid with category filters
- **Selections**: Card-based selection view with large images
- Side-by-side option comparison
- One-tap confirm/decline selection
- Budget impact prominently displayed
- **Draws**: Draw list with status badges (pending, approved)
- Draw detail with line item breakdown
- Approve/Request Changes with confirmation modal
- Push notifications for new draws or selections requiring action
- Biometric login (Face ID/Touch ID) for security
- Offline: View cached photos and approved selections
- Progressive web app (PWA) support for home screen install
